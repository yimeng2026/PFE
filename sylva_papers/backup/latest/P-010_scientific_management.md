## 摘要

本文提出了一个面向大型研究项目的科学问题库管理框架。该框架通过系统化的分类、排序和跟踪机制，帮助研究团队有效管理复杂项目中的大量问题。框架采用九级优先级分类（P0-P8），基于影响度、紧迫性、工作量和风险的综合评分算法进行科学排序。本文详细描述了框架的设计原理、实施方法和应用案例，并讨论了其在Sylva项目中的实际应用效果。

**关键词**: 问题库管理；科学方法论；优先级排序；项目管理；知识管理；大型研究项目

## 1. 引言

### 1.1 大型研究项目的管理挑战

现代科学研究项目，特别是跨学科、长周期的基础理论研究，面临独特的管理挑战：

- **问题数量庞大**: 一个大型项目可能涉及数百个待解决问题
- **依赖关系复杂**: 问题之间存在复杂的依赖和阻塞关系
- **优先级动态变化**: 随着研究进展，问题的优先级不断变化
- **知识沉淀困难**: 研究过程中的经验和教训难以系统化保存

### 1.2 现有方法的局限

| 方法 | 优点 | 局限 |
|------|------|------|
| **待办清单** | 简单直观 | 无法处理复杂依赖 |
| **甘特图** | 可视化进度 | 不适合研究型项目 |
| **敏捷开发** | 快速迭代 | 不适合基础理论 |
| **问题跟踪系统** | 系统化记录 | 缺乏科学排序 |

### 1.3 科学问题库框架

本文提出的框架通过以下创新解决上述问题：
1. **九级分类**: 从紧急修复到战略研究的完整覆盖
2. **科学排序**: 基于多因素的综合评分算法
3. **依赖追踪**: 显式记录问题间的依赖关系
4. **知识沉淀**: 自动记录决策过程和失败经验

## 2. 框架设计

### 2.1 九级优先级分类

问题库采用九级分类结构：

| 级别 | 名称 | 定义 | 响应时间 |
|------|------|------|----------|
| **P0** | 紧急修复 | 阻塞性、系统崩溃级问题 | 立即 |
| **P1** | 核心证明 | 关键定理、核心算法 | 24小时 |
| **P2** | 理论扩展 | 新方向、新应用 | 1周 |
| **P3** | 数值验证 | 实验检验、模拟验证 | 2周 |
| **P4** | 工具开发 | 基础设施、开发工具 | 1月 |
| **P5** | 文档完善 | 论文写作、文档更新 | 按需 |
| **P6** | 代码重构 | 质量提升、技术债务 | 按需 |
| **P7** | 外部集成 | 系统对接、数据交换 | 按需 |
| **P8** | 战略研究 | 长期目标、探索性研究 | 长期 |

### 2.2 综合评分算法

问题优先级由以下公式计算：

$$PriorityScore = \frac{Impact \times Urgency}{Effort + Risk}$$

其中：
- **Impact** (1-10): 对项目目标的贡献度
  - 10: 决定项目成败
  - 7-9: 重大突破
  - 4-6: 显著改进
  - 1-3: 边际改进
  
- **Urgency** (1-10): 时间紧迫性
  - 10: 立即处理，否则项目阻塞
  - 7-9: 近期必须完成
  - 4-6: 中期规划
  - 1-3: 长期目标
  
- **Effort** (人月): 估计工作量
  - 使用对数尺度: 0.1, 0.5, 1, 2, 3, 6, 12, 24, 48
  
- **Risk** (1-10): 失败风险
  - 10: 几乎必然失败
  - 7-9: 高风险
  - 4-6: 中等风险
  - 1-3: 低风险

### 2.3 依赖关系管理

问题之间的依赖关系显式记录：

```
问题A [P1] ──depends_on──→ 问题B [P0]
    │
    └──blocks──→ 问题C [P2]
```

**依赖规则**:
1. 被依赖的问题优先级自动提升
2. 循环依赖检测和报警
3. 依赖链长度限制（最大5层）

## 3. 实施方法

### 3.1 问题录入

每个问题包含以下字段：

```json
{
  "id": "P1-001",
  "title": "evalNode终止证明",
  "description": "证明evalNode函数的well-foundedness",
  "category": "P1",
  "impact": 9,
  "urgency": 8,
  "effort": 2,
  "risk": 4,
  "priority_score": 12.0,
  "dependencies": ["P0-003"],
  "blocks": ["P1-002", "P1-003"],
  "status": "in_progress",
  "assignee": "team_alpha",
  "created": "2026-04-01",
  "updated": "2026-04-22",
  "notes": "使用强归纳法框架"
}
```

### 3.2 排序与调度

**每日排序流程**:
1. 计算所有开放问题的PriorityScore
2. 按分数降序排列
3. 检查依赖关系，调整顺序
4. 分配资源（人力、计算资源）
5. 更新进度

**资源分配原则**:
- P0-P1: 立即分配全部必要资源
- P2-P3: 按优先级分配
- P4-P8: 在核心任务完成后处理

### 3.3 进度跟踪

**度量指标**:

| 指标 | 定义 | 目标 |
|------|------|------|
| **完成率** | 已完成问题/总问题 | >80% |
| **平均解决时间** | 问题创建到关闭的平均时间 | <30天 |
| **阻塞率** | 被阻塞问题/总开放问题 | <10% |
| **优先级漂移** | 优先级变更次数/问题 | <2次 |

## 4. 应用案例：Sylva项目

### 4.1 项目概况

Sylva项目是一个跨学科基础理论研究项目，涉及数学、物理、计算机科学等领域。

**规模**:
- 总问题数: 57
- 核心模块: 11
- 参与人员: 5+ Agent
- 时间跨度: 6个月

### 4.2 问题库统计

| 级别 | 数量 | 已完成 | 进行中 | 待启动 |
|------|------|--------|--------|--------|
| P0 | 3 | 2 | 1 | 0 |
| P1 | 8 | 3 | 3 | 2 |
| P2 | 12 | 2 | 4 | 6 |
| P3 | 10 | 1 | 3 | 6 |
| P4 | 8 | 2 | 2 | 4 |
| P5 | 6 | 1 | 2 | 3 |
| P6 | 4 | 0 | 1 | 3 |
| P7 | 3 | 0 | 0 | 3 |
| P8 | 3 | 0 | 0 | 3 |
| **总计** | **57** | **11** | **16** | **30** |

### 4.3 关键成果

1. **编译通过率**: 从40%提升到100%
2. **证明完整性**: 从30%提升到75%
3. **问题关闭率**: 每月平均关闭8个问题
4. **优先级调整**: 平均每个问题调整1.2次

## 5. 讨论

### 5.1 优势

1. **系统化**: 将模糊的管理任务转化为可量化的流程
2. **透明性**: 所有问题状态可见，便于协作
3. **适应性**: 优先级动态调整，适应研究变化
4. **知识沉淀**: 自动记录决策过程和失败经验

### 5.2 局限

1. **量化困难**: 某些因素（如创新性）难以量化
2. **过度优化**: 可能过度关注短期目标，忽视长期探索
3. **依赖复杂性**: 复杂依赖关系可能导致排序困难
4. **人员因素**: 未考虑人员技能和偏好

### 5.3 未来改进

1. **机器学习**: 使用历史数据预测解决时间
2. **自动分配**: 基于技能匹配自动分配问题
3. **情感分析**: 分析问题描述的情感倾向
4. **可视化**: 开发交互式问题库可视化工具

## 6. 结论

### 6.1 核心贡献

1. **九级分类**: 建立了完整的问题优先级分类体系
2. **科学排序**: 提出了基于多因素的综合评分算法
3. **依赖管理**: 实现了显式的依赖关系追踪
4. **知识沉淀**: 建立了系统化的知识管理机制

### 6.2 理论意义

科学问题库管理框架表明：
- 大型研究项目需要系统化的管理方法
- 量化管理可以提高研究效率
- 知识沉淀是持续改进的基础
- 动态适应是研究管理的本质
