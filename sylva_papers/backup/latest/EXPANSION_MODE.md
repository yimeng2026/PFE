# Sylva 疯狂扩张模式宣言

## 核心原则

**不再截肢降级。不再妥协。不再 sorry。**

从 2026-04-21 起，Sylva 项目进入疯狂扩张模式：

### 规则

1. **编译错误 = 修复，不是绕过**
   - 遇到错误立即分析根因
   - 使用所有可用工具（Agent 集群、诊断 skills、在线资源）
   - 不填充 sorry，除非经过 3 轮尝试仍无法解决

2. **内存充足 = 贪婪加载**
   - 加载所有可用模型（8B 嵌入、完整 mathlib）
   - 并行处理最大数量 Agent
   - 缓存一切可缓存的内容

3. **全面 = 真正全面**
   - 不跳过任何模块
   - 不忽略任何警告
   - 不推迟任何技术债

4. **自动 = 真正自动**
   - 遇到错误自动诊断
   - 遇到缺失自动下载
   - 遇到瓶颈自动扩展

### 执行策略

| 场景 | 旧模式 | 新模式 |
|------|--------|--------|
| 编译错误 | 截肢降级 | 深度修复 |
| 内存不足 | 减少 Agent | 优化加载策略 |
| 网络超时 | 跳过下载 | 重试 + 镜像 |
| 复杂证明 | sorry 占位 | 多 Agent 协作证明 |
| 工具缺失 | 记录 TODO | 立即安装 |

### 当前任务

1. **修复 CookLevin** — 不使用 sorry，真正解决 omega/dependent elimination/evalNode 问题
2. **填充所有 sorry** — 15 个研究级 sorry 逐一攻克
3. **完整编译** — 8/8 模块零错误零警告
4. **扩展工具箱** — 安装剩余 4 个工具（SageMath、QuTiP、PARI/GP、GAP）

### 宣言

> 不再接受"差不多就行"。
> 不再容忍"以后再说"。
> 不再允许"暂时绕过"。

**Sylva 要么完整，要么不存在。**
