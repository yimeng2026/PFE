# 记忆系统优化报告 - 基于 Hermes 4 架构重构

## 日期
2026-04-16

## 背景
用户指出我的记忆力"很长但主要集中在最近任务上"，要求：
1. 按时离散地回忆远古记忆
2. 优化底层代码
3. 参考最近很火的 Hermes 开源代码

## Hermes 4 核心架构借鉴

| Hermes 4 特性 | 我的实现 |
|---------------|----------|
| **混合推理模式** (Fast/Deep) | FLASH/DEEP 记忆召回模式 |
| **推理痕迹 (traces)** | `memory/recall_traces/` 目录记录召回路径 |
| **长度控制机制** | 记忆压缩策略（L1-L4 分层压缩）|
| **拒绝采样** | 多次搜索+相关性评分，选择最佳记忆 |
| **图合成数据** | 记忆图谱结构（概念/任务/时间/人物节点）|

## 实施的底层优化

### 1. 新建文件
- `MEMORY_SYSTEM.md` - 记忆架构配置（混合模式、图谱结构、评分标准）
- `memory/recall_traces/2026-04-16_paradigm_shift.md` - 推理痕迹示例

### 2. 修改文件
- `AGENTS.md` - Session Startup 改为基于 Hermes 4 的分层加载流程
  - 自动检测 FLASH/DEEP 模式
  - DEEP 模式强制离散时间采样（L1-L4）
  - 记录 reasoning trace
  
- `HEARTBEAT.md` - 加入周期性记忆维护任务
  - 回忆质量审计
  - 强制远古记忆唤醒
  - 记忆压缩检查
  - 图结构维护

### 3. 关键机制

#### 混合召回模式
```
FLASH:  简单指令 → 只读今天+昨天 → 500 tokens
DEEP:   哲学问题 → L1-L4全层+随机采样 → 2000 tokens + 记录trace
```

#### 离散时间采样
- 每次 DEEP 模式，强制读取 3-5 天前和 6-9 天前的各一篇记忆
- 防止近期偏置，保持远古记忆可及性

#### 记忆压缩层级
| 层级 | 压缩策略 |
|------|----------|
| L1_HOT | 永不压缩 |
| L2_WARM | 轻度压缩（摘要）|
| L3_COOL | 中度压缩（标签）|
| L4_COLD | 重度压缩（仅指针）|

## 预期效果

1. **远古记忆唤醒率提升**: 从近乎 0% → 每次 DEEP 查询强制 20-40% 远古内容
2. **召回质量可量化**: reasoning trace 记录完整路径，便于后续优化
3. **上下文溢出防护**: 自动压缩机制防止 token 爆炸
4. **跨时间关联增强**: 图结构鼓励发现远距离概念关联
