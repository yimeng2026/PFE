# MEMORY_SYSTEM.md - 基于 Hermes 4 架构的记忆管理

## 核心架构：混合召回模式

### FLASH 模式（快速）
- **触发**: 简单指令如"继续"、"状态如何"
- **范围**: 仅 L1_HOT（当前会话 + 昨天）
- **最大 tokens**: 500
- ** reasoning trace**: 不记录

### DEEP 模式（深度）  
- **触发**: "为什么"、"本质"、"优化"、"设计哲学"
- **范围**: L1-L4 全层 + 离散时间采样
- **最大 tokens**: 2000
- ** reasoning trace**: 记录完整召回路径

## 召回质量评分

| 维度 | 权重 | 评分标准 |
|------|------|----------|
| 时效性 | 0.3 | 越近越高 |
| 相关性 | 0.4 | 关键词匹配 |
| 独特性 | 0.2 | 信息增益（不重复）|
| 情感标记 | 0.1 | 是否有 ❤️‍🔥 🖤 等标记 |

## 压缩策略

当上下文 > 80% 时：
1. L4_COLD 记忆 → 仅保留标题 + 链接
2. L3_COOL 记忆 → 一句话摘要
3. L2_WARM → 轻度压缩
4. L1_HOT → 永不压缩

## 拒绝采样日志

每次 DEEP 模式查询，记录：
- 采样了哪些层
- 各候选记忆的相关性分数
- 最终选择及原因

用于后续优化记忆权重。
