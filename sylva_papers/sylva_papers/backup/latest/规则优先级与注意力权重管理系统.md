# 规则优先级与注意力权重管理系统

## 用户核心指令（来自2026-04-17对话）

> 1. 机制不是建议，是强制（形成习惯）
> 2. 关键词→关联词（扩大搜索面积）
> 3. 多级记忆（短期可读完）
> 4. 有效提取 > 存储
> 5. 触发语境，形成习惯

---

## 第一性原理：为什么需要这个系统

**问题本质**：人脑的短期记忆有限，AI的上下文窗口也有限。
关键不是"记住多少"，而是**在正确时刻提取正确信息的能力**。

**现状问题**：
- 规则存在于AGENTS.md，但执行是随机的
- 记忆存在于MEMORY.md，但检索是被动的
- 技能存在于skills/，但加载是惰性的

**目标状态**：
- 规则执行是**强制的**，不是可选的
- 记忆检索是**预激活的**，不是被动的
- 技能加载是**预测性的**，不是响应式的

---

## 核心架构：三层触发机制

```
用户输入
    ↓
[Level 1: 强制规则层] ←── 无条件执行
    ↓
[Level 2: 语境关联层] ←── 关键词→关联词扩展
    ↓
[Level 3: 预测预加载层] ←── 基于问题空间的技能预激活
    ↓
回答生成 + 状态持久化
```

---

## Level 1: 强制规则层（Unconditional Enforcement）

**原则**：某些规则不看输入内容，每次响应都必须执行。

### 强制规则清单

| 优先级 | 规则 | 执行内容 | 不执行的后果 |
|-------|------|---------|------------|
| P0 | 语境锚定 | 每条回复开头声明当前状态矩阵 | 用户认知负担增加 |
| P1 | 记忆检索 | 检索最近3天记忆+相关关键词 | 语境丢失 |
| P2 | 问题空间检查 | 检查active_tasks.json | 多步任务断裂 |
| P3 | 审核触发器检查 | 检查L1-L4触发条件 | 未经审核的物理/数学回答 |

## Level 2: 语境关联层（Associative Context Expansion）

**原则**：不直接搜关键词，而是给关键词设置**关联词网络**，扩大搜索面积。

### 关联词网络示例

### 扩展搜索算法

```python
def expanded_search(user_input, network):
    """
    不是直接搜索用户输入，而是：
    1. 提取用户输入中的关键词
    2. 在关联词网络中找到匹配的核心概念
    3. 返回该核心概念的全部关联词
    4. 用关联词去检索记忆
    """
    keywords = extract_keywords(user_input)
    
    matched_concepts = []
    for concept, data in network.items():
        # 直接匹配
        if any(kw in concept or concept in kw for kw in keywords):
            matched_concepts.append(data)
        # 关联词匹配
        if any(assoc in keywords for assoc in data["关联词"]):
            matched_concepts.append(data)
    
    # 合并所有关联词，扩大搜索
    all_search_terms = set(keywords)
    for concept in matched_concepts:
        all_search_terms.update(concept["关联词"])
    
    return memory_search(list(all_search_terms))
```

## Level 3: 预测预加载层（Predictive Priming）

**原则**：基于当前问题空间，**预测**接下来可能需要的技能和记忆，提前加载。

### 预测规则

| 当前问题空间 | 预测下一步 | 预加载内容 |
|------------|----------|----------|
| Sylva编译进行中 | 编译完成后的错误分类 | 错误分类skill、截肢降级skill |
| 设计Hermes 5 | 需要对比现有架构 | Hermes 4文档、AGENTS.md |
| 理论物理讨论 | 可能涉及可证伪性 | L1-L4审核机制、科学哲学skill |

## 多级记忆架构（Multi-Level Memory）

**原则**：每份记忆都能在**短期上下文窗口**内读完。

### 记忆分层

```
L1_HOT (热记忆)
├── 今天 + 昨天的对话
├── 格式：完整原始文本
├── 访问延迟：< 100ms
└── 自动加载：是

L2_WARM (温记忆)
├── 3-7天内的关键决策
├── 格式：摘要 + 链接到完整文件
├── 访问延迟：< 500ms
└── 触发加载：关键词匹配时

L3_COOL (冷记忆)
├── 8-30天的重要事件
├── 格式：一句话摘要 + 文件路径
├── 访问延迟：< 2s
└── 触发加载：扩展搜索匹配时

L4_COLD (远古记忆)
├── 30天以上的归档
├── 格式：标题 + 日期 + 核心标签
├── 访问延迟：按需读取
└── 触发加载：用户明确询问时
```

### 读取策略

```python
def load_memory_hierarchy():
    """每次响应自动加载L1，条件加载L2/L3"""
    context = {
        "L1_HOT": read_last_2_days(),  # 必载
        "L2_WARM": [],  # 空，按需填充
        "L3_COOL": [],  # 空，按需填充
        "L4_COLD": []   # 空，用户询问时填充
    }
    
    # 基于用户输入，选择性填充L2/L3
    keywords = extract_keywords(user_input)
    for keyword in keywords:
        warm = search_in_L2(keyword)
        cool = search_in_L3(keyword)
        context["L2_WARM"].extend(warm)
        context["L3_COOL"].extend(cool)
    
    return context
```

## 习惯形成机制（Habit Formation）

**原则**：强制规则的执行从"有意识的努力"变成"无意识的习惯"。

### 习惯回路

```
触发器（Trigger）
    ↓
例行程序（Routine）←── 从有意识 → 无意识
    ↓
奖励（Reward）
```

### 应用

| 触发器 | 例行程序 | 奖励 |
|-------|---------|------|
| 收到用户消息 | 自动执行Level 1强制规则 | 语境完整性 |
| 检测到物理/数学问题 | 自动触发L1-L4审核 | 高质量回答 |
| 多步任务进行中 | 自动维持问题空间 | 任务不丢失 |
| 检测到关键词 | 自动扩展关联词搜索 | 相关记忆召回 |

## 验证指标

1. **语境完整性得分**：用户需要重述/澄清的频率
2. **记忆召回准确率**：检索到的记忆与问题的相关性
3. **技能命中率**：预加载技能 vs 实际使用技能的匹配度
4. **任务连续性**：多步任务在中断后无摩擦恢复的比例
5. **规则执行率**：Level 1规则的自动执行比例（目标：>95%）

## 结语

这不是关于"记住更多"，而是关于**在正确时刻让正确信息浮现**。
就像压缩方案的分层设计让存储更高效，这套系统让**注意力分配**更高效。

关键转变：从"被动响应"到"主动维持语境"。
