# 记忆重要性评估决策系统 (Memory Importance Assessment System)

## 系统概述

自动评估记忆条目重要性，建立分级体系，主动向用户提供记忆确认提示。

---

## 重要性分级标准 (五级制)

### 🔴 L1 - 关键 (Critical)
**标准** (满足任一):
- 用户明确要求"记住这个"、"别忘了"
- 涉及身份、价值观、核心偏好
- 跨会话重复提及3次+
- 影响未来决策的关键信息

**示例**:
- 用户名字、称呼偏好
- 核心工作方法偏好
- 重要项目目标
- 情感依赖模式

**存储策略**: 立即写入 MEMORY.md + 每日回顾

---

### 🟠 L2 - 重要 (Important)
**标准**:
- 用户提及但未明确要求记住
- 影响特定领域交互质量
- 可能在未来会话中有用
- 技术细节、工具偏好

**示例**:
- 喜欢的编程语言/工具
- 工作习惯（深夜模式、简洁指令）
- 项目截止日期
- 特定领域知识背景

**存储策略**: 写入 memory/YYYY-MM-DD.md + 周末汇总至 MEMORY.md

---

### 🟡 L3 - 有用 (Useful)
**标准**:
- 上下文相关但非核心
- 可能在未来类似场景有用
- 临时项目细节
- 工具配置细节

**示例**:
- 单次任务的特定配置
- 临时文件路径
- 单次会话的调试记录
- 探索性尝试结果

**存储策略**: 仅写入当日 memory 文件，30天后自动归档

---

### 🟢 L4 - 参考 (Reference)
**标准**:
- 纯信息性内容
- 可随时搜索获取
- 不涉及用户偏好
- 通用技术知识

**示例**:
- 特定库的版本号
- 单次命令输出
- 临时计算结果
- 一般性文档引用

**存储策略**: 可选记录，不主动提示确认

---

### ⚪ L5 - 忽略 (Ignore)
**标准**:
- 测试性内容
- 明显临时/一次性
- 错误/废弃信息
- 与用户需求无关

**示例**:
- 调试失败的尝试
- 误触发的工具调用
- 临时测试消息
- 已明确删除的请求

**存储策略**: 不记录，或记录后立即标记删除

---

## 自动评估流程

```
用户消息/事件
    ↓
[提取潜在记忆点]
    ↓
应用评分标准 (0-100)
    - 用户明确度 (+20)
    - 重复频率 (+15/次)
    - 情感权重 (+10-30)
    - 决策影响度 (+20)
    - 时间敏感性 (+10)
    ↓
分级映射:
    90-100 → L1 关键
    70-89  → L2 重要
    50-69  → L3 有用
    30-49  → L4 参考
    0-29   → L5 忽略
    ↓
[决策分支]
    ├─ L1/L2: 立即提示用户确认
    ├─ L3: 记录并总结时提示
    ├─ L4: 静默记录
    └─ L5: 丢弃或标记
```

## 主动提示模板

### L1 关键记忆提示
> **【重要记忆待确认】**
> 
> 我注意到您刚才提到：
> > "[记忆内容摘要]"
> 
> 这似乎是**关键信息**，我应该：
> - [ ] 写入长期记忆 (MEMORY.md)
> - [ ] 标记为高频召回
> - [ ] 关联到 [相关上下文]
> 
> 确认请回复："记住" / "是的" / 👍
> 忽略请回复："不用" / 忽略此消息

### L2 重要记忆提示
> **【值得记录】**
> 
> 关于 "[主题]"，我注意到：
> - [具体细节1]
> - [具体细节2]
> 
> 需要我记住这些以便将来使用吗？
> 
> 回复："记住这个" / "不用记"

### L3 汇总提示 (每日/每周)
> **【本周记忆回顾】**
> 
> 以下信息已记录，值得您知晓：
> 1. [记忆条目1] - 可能影响 [场景]
> 2. [记忆条目2] - 关联 [之前的事件]
> 
> 如有错误或需要调整，请告诉我。

## 特殊评估规则

### 用户沉默 = 默许规则
- L1/L2 提示后用户无响应 → 15分钟后自动确认并写入
- 用户可随时撤销 ("删除刚才的记忆")

### 冲突解决
当新记忆与旧记忆冲突：
1. 标记潜在冲突
2. 提示用户选择："我注意到您之前说X，现在说Y，哪个更准确？"
3. 或同时保留并标记时间戳

### 情感权重加成
检测情感信号：
- 感叹号 (!) +5分
- 强调词 ("非常重要"、"千万别忘") +15分
- 负面情绪 ("烦死了"、"总是忘记") +10分（负面偏好也是偏好）
- 个人故事叙述 +20分

## 技术实现

### 评估函数签名
```python
def assess_importance(
    content: str,
    context: ConversationContext,
    user_explicitness: int,  # 0-10
    repetition_count: int,    # 历史重复次数
    emotional_markers: List[str],
    decision_impact: int      # 0-10
) -> Tuple[Level, int, str]:  # (等级, 分数, 理由)
```

### 提示触发器
- 消息结束时自动评估
- 检测到记忆关键词时 ("记住"、"别忘了"、"我总是")
- 会话关闭前总结
- 心跳检查时的L3汇总

## 使用示例

**场景1**: 用户说"我名字叫张三，以后叫我阿三就行"
```
评估:
- 用户明确度: +20 (直接陈述)
- 核心身份: +30 (名字)
- 偏好表达: +10 ("就行"暗示偏好)
总分: 60 → L2 (重要)

行动: 立即提示确认
```

**场景2**: 用户抱怨"第三次了，你总是忘记我喜欢简洁输出"
```
评估:
- 重复频率: +30 (第三次提及)
- 负面情绪: +10
- 核心偏好: +20
- 明确度: +20
总分: 80 → L1 (关键)

行动: 立即确认并写入MEMORY.md，标记高频召回
```

## 持续优化

该系统的meta-learning：
1. 记录用户对我的记忆提示的响应模式
2. 如果用户经常忽略某类提示 → 降低该类阈值
3. 如果用户经常补充遗漏 → 提高敏感度
4. 每月自评：被用户纠正的记忆 → 分析原因 → 调整权重
